# estack 项目总览

## 一句话定位

estack 是一个面向企业的 **AI-native innovation skill set**，帮助企业重新设计人和 AI 在创新与知识工作中的协作方式。

对外说法：企业 AI-native innovation workflow 工具箱。

底层逻辑：人机协作配置如何影响创造性问题解决、创新产出、采用与治理。

## GitHub

- Repo：<https://github.com/tuoyiwen/estack>
- 当前版本：v0.1
- 当前状态：已建立 1 个中枢 skill + 6 个核心 skill

## 合作者给出的当前任务

> 添加知识库，优化 onboarding 体验，规划产出物料

这意味着 v0.2 的重点不是继续增加 skill 数量，而是让 estack 更像一个可以被新用户理解、安装、试用和转述的产品。

## 当前核心 skills

| Skill | 对外名称 | 主要输出 |
|---|---|---|
| `$estack` | Estack router | 推荐使用路径和下一步 skill |
| `$e-innovation-map` | AI Innovation Map | 企业 AI 创新场景地图和优先级 |
| `$e-collaboration-design` | Collaboration Design | 人机协作配置和交互协议 |
| `$e-innovation-workflow` | Innovation Workflow | AI-native 工作流蓝图和 SOP |
| `$e-innovation-pilot` | Innovation Pilot | 30/60/90 天试点方案 |
| `$e-innovation-metrics` | Innovation Metrics | 创新效果、采用与互动指标 |
| `$e-ai-governance` | AI Governance | 治理矩阵、复核规则和风险边界 |

## 关键原则

- 对外使用 innovation / collaboration 语言，不把 creativity 放在门面上。
- 不做泛泛 AI 工具推荐器，而做企业创新工作方式重构器。
- 先让用户跑通一条主线：map -> collaboration design -> workflow -> pilot -> metrics -> governance。
- 所有 skill 都要有明确输入、步骤、输出物和不该做什么。
- 论文机制保留在底层：人机配置、问题定义顺序、新颖性/有用性、采用与治理。
