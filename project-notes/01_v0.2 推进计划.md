# v0.2 推进计划

## 今日目标

把 estack 从 6 个 skill 草案升级成一个新用户能看懂、能安装、能试用、能转述给合作者的 v0.2 版本。

## 任务 1：添加知识库

- [ ] 建立 estack 的基础知识库结构：`references/` 或 `knowledge-base/`。
- [ ] 补充 4 类核心知识：企业 AI innovation use case、human-AI collaboration patterns、innovation metrics、AI governance rules。
- [ ] 为每个核心 skill 增加可复用参考材料，而不是把所有内容塞进 `SKILL.md`。
- [ ] 先做最小知识库，不追求百科全书：每类 1 页，能支撑用户第一次试用即可。

## 任务 2：优化 onboarding 体验

- [ ] 重写 README 的 first 5 minutes：用户安装后第一步该运行哪个 skill。
- [ ] 增加 3 个示例 prompt：企业找 AI 创新场景、人机协作配置设计、AI innovation pilot 设计。
- [ ] 明确推荐路径：`estack` -> `e-innovation-map` -> `e-collaboration-design` -> `e-innovation-pilot`。
- [ ] 加一个“如果你是企业管理者 / 顾问 / 研究者，分别怎么用”的入口说明。

## 任务 3：规划产出物料

- [ ] 产出一个 GitHub 首页版 README：定位、核心 skills、安装方式、示例用法。
- [ ] 产出一个 one-page overview：estack 是什么、解决什么问题、6 个 skill 分别输出什么。
- [ ] 产出一个 demo case：例如“新产品概念开发”或“营销创意流程”如何用 estack 跑一遍。
- [ ] 产出一组示例输出模板：Use Case Portfolio、Collaboration Configuration Card、Pilot Charter、Metrics Dashboard、Governance Matrix。

## 验收标准

- [ ] 一个不了解 estack 的人，打开 GitHub 后 3 分钟内能知道它是做什么的。
- [ ] 一个企业用户能直接复制示例 prompt 试用。
- [ ] 每个 skill 都有明确输入、步骤、输出物和不该做什么。
- [ ] 对外命名保持 innovation / collaboration，不把 creativity 放在门面上。
- [ ] 论文机制仍然保留在底层：人机配置、问题定义顺序、新颖性/有用性、采用与治理。

## 今天优先顺序

1. 先改 README 和 onboarding。
2. 再补知识库最小结构。
3. 最后做 demo case 和输出模板。
