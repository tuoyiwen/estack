# Demo Case 设计

## 推荐 demo：新产品概念开发

选择原因：

- 企业容易理解。
- 能覆盖问题定义、信息搜索、创意生成、筛选、整合、评估。
- 能体现人机配置差异，而不是只展示 AI 生成文本。
- 可以自然连接创新指标：新颖性、有用性、可行性、战略匹配。

## Demo 背景

一家 B2B SaaS 公司想探索下一代 AI 功能，但团队不确定应从客户痛点、竞品趋势、内部技术能力，还是 AI 自动生成方向开始。

## 使用路径

1. `$e-innovation-map`：识别产品创新场景，选出“新产品概念开发”。
2. `$e-collaboration-design`：比较 human-first 与 AI-first 的问题定义顺序。
3. `$e-innovation-workflow`：设计 AI-native 产品概念开发流程。
4. `$e-innovation-pilot`：设计 30/60/90 天试点。
5. `$e-innovation-metrics`：定义新颖性、有用性、采用与业务指标。
6. `$e-ai-governance`：定义客户数据、竞品资料和产品路线图的使用边界。

## Demo 输出物

- AI Innovation Use Case Portfolio
- Human-AI Collaboration Configuration Card
- AI-native Workflow Blueprint
- Pilot Charter
- Innovation Metrics Dashboard
- AI Governance Matrix
