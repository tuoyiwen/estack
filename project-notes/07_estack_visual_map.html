User Journey And Skill Path

The main path converts a broad enterprise AI request into a bounded pilot decision. The decision variable at the end is scale, iterate, or stop.

recommended flow
0Enterprise AI requestExample object: an innovation or knowledge-work workflow where the team asks where AI should enter.
→
1$estackClassifies the request and routes the next action.
→
2$e-innovation-mapClassifies use cases as Now, Next, Later, or Avoid.
3$e-collaboration-designDefines human role, AI role, authority, review, and escalation.
→
4$e-innovation-workflowConverts the configuration into an as-is/to-be workflow and SOP.
→
5$e-innovation-pilotDefines a bounded 30/60/90-day experiment.
6a$e-innovation-metricsDefines baseline, dashboard spec, threshold, and decision rule.
+
6b$e-ai-governanceDefines data, authority, review, records, and accountability.
→
7Scale / iterate / stopDecision object: pilot result. Decision variable: evidence against the pre-defined rule.

Artifact Chain

Each skill produces a concrete object that becomes input for the next skill. This chain is the core operating model of estack.

deliverables
AI Innovation Use Case Portfolio

Object: candidate enterprise innovation tasks.

Decision variable: Now / Next / Later / Avoid.

Collaboration Configuration Card

Object: selected task.

Variables: sequence, AI role, human role, authority, safeguards.

Human-AI Interaction Protocol

Object: task episode.

Variables: briefing format, AI response structure, review checklist, escalation rule.

AI-native Workflow Blueprint

Object: operating workflow.

Variables: human work, AI work, handoff, decision, evidence, risk control.

New SOP + Implementation Backlog

Object: repeatable operating model.

Variables: owner, input, review standard, dependency, priority.

AI Innovation Pilot Charter

Object: bounded pilot.

Variables: owner, participants, comparison design, success criteria, stop criteria.

Metrics Dashboard Spec

Object: pilot evidence system.

Variables: metric, definition, data source, cadence, threshold, owner.

Governance Matrix

Object: workflow risk controls.

Variables: risk tier, data allowed, AI authority, human review, approval, records.

Scale / Iterate / Stop Memo

Object: pilot decision.

Variable: evidence against the decision rule.
