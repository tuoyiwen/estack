# 企业面临的 Token 不经济问题（原文）

> 原文归档。定价数字与厂商事件时效性强，抽取时 evidence strength 标 partial；文中引用的三篇 2026 论文（Salim Tokenomics、Gao SkillReducer、Han SWE-Skills-Bench）为 direct 证据，已另建 source card。抽取判断见 [[../source-triage/2026-07-05|2026-07-05 triage]]。

## 摘要（bestblogs AI 摘要）

本文以微软收回 Claude Code 许可、Uber 耗尽全年 AI 预算等事件为引，提出「Token 不经济」概念：供给端头部模型厂商通过分层定价抬高 token 价格，中低端模型量价齐升；技术端上下文陷阱、分词器黑箱、技能无效调用、多 Agent 沟通税与长程熵增等结构性损耗呈指数级叠加；应用端 token 的有效场景高度局限于编程等数字化领域，向物理世界跨越面临非对称验证成本的巨大障碍。产业链风险向中游模型厂商聚集，并通过循环融资与影子信贷形成潜在泡沫。解决方案从上下文压缩、技能优化、模型路由、预算硬约束等技术侧，以及成本纪律、配给制、场景发现等商业侧入手，回归 ROI。

## 原文

### 引子

近期部分媒体爆出微软收回内部 Claude Code 许可。Claude Code 在微软内部开放仅 6 个月就成为最受欢迎的辅助开发软件之一，随之而来的是 token 消耗剧增、成本暴涨，但产出质量不尽如人意。微软踩下刹车，将员工导向自家的 Copilot CLI。

Token 消耗与实际产出不成比例的现象在其他平台企业也普遍存在：Uber 仅用 4 个月就耗尽了 2026 全年 AI 编程工具预算；亚马逊部分员工无意义消耗 token；Meta 悄悄撤下内部员工的 Tokenmaxxing 排行榜，不再鼓励无产出的 token 消耗。人人都在拥抱 AI，但还没有找到正确的姿势；企业都在强调 AI 原生，但（暂时）没看到收益，只看到越来越长的账单。作者称之为"token 不经济"。

Token 不经济是企业内部管控不力、token 使用回报有限、Agent 本身的架构设计（Skill 重复调用、长程任务内耗、多智能体协同成本）等多重因素叠加的结果。

### 一、好货不便宜（供给端）

**领导者的分层策略**：Anthropic 最早认识到编程是 token 变现核心场景，自 Claude 3 起采用旗舰-中端-轻量立体产品组合分层定价（Opus $15/$75、Sonnet $3/$15、Haiku $1/$5），后续通过 Mythos Preview（$25/$125）在 Opus 上置入超高端分层，Fable 5 以安全约束松紧为由采用 $10/$50，形成能力分层、风险分层、定价分层的三维定价策略。ARR 从 2024 年底约 10 亿美元升至 2026 年 5 月约 450 亿美元。

**追赶者的价格拉扯**：OpenAI 砍掉 Sora 等项目重新聚焦编码与 Agent，旗舰高价对标（GPT 5.5 $5/$30 vs Opus 4.7/4.8 $5/$25），轻量低价抢份额（mini/nano 大幅低于 Haiku）。Google 受生态牵制更谨慎，Gemini Pro 定价始终低于同期竞品，Flash-Lite 压到开源模型价位以价换量。

**次级/轻量与开源/半开源市场在需求爆发中默默涨价**：表面卷价格（DeepSeek、Qwen、GLM 以旗舰定位、轻量定价切入），但每代产品都抬升价格地板：Haiku 3.5 -> 4.5 上浮 20%；GPT mini 系列翻番；Gemini Flash 输出价翻 6 倍多；GLM-5 较 4.7 提价 67%-100%。根本原因是经济型 token 消费量爆炸式增长，竞争逻辑从"哪家更便宜"转向"哪家性价比更高"。

整体格局：高端定价固化、中端量价齐升、经济型跟随追涨。与模型厂商收入爆发相对应的，必然是下游 token 使用者的成本攀升。

### 二、智能体的隐形消耗（技术端）

**上下文陷阱**：Agent 架构天然放大长文本消耗，历史记录被反复读取、反复计费。Salim et al. (2026) 对 ChatDev 框架的分析发现 Code Review 阶段平均消耗总 token 的 39.5%，近四成花在 Agent 之间反复传递已有信息。

**分词器黑箱**：闭源模型分词器是黑箱，更新换代伴随 token 密度变化。Anthropic Opus 4.7 更换分词器后官方口径膨胀 1.0-1.35 倍，独立实测更高（Finout 企业 prompt 加权 +47%；ClaudeCodeCamp 七类文件平均 +32.5%；Simon Willison 实测系统提示词 +46%、高分辨率图片 +201%）。更细粒度分词是否提升性能，缺乏系统性公开论证，近 50% 成本增幅是否合理是悬而未决的问题。

**技能的无意义调用**：Gao et al. (2026) 对 55,315 个公开技能的实证：26.4% 完全没有路由描述；超过 60% 的正文是背景解释而非可执行规则；部分技能单次调用注入数万乃至十余万 token。Han et al. (2026) SWE-Skills-Bench：49 个公开软件工程技能中 39 个（79.6%）没有带来通过率任何提升，平均效用增量仅 1.2 个百分点，而 token 开销最高增加 451%；仅 7 个编码特定领域专业知识的技能带来有意义提升（最高 +30pp）；3 个技能因版本冲突导致性能下降。技能效用高度依赖场景匹配度，盲目调用徒增成本。

**多 Agent 的废话与长任务的跑偏**：多 Agent 沟通税（重复任务背景、已有结论、格式化套话）；长程任务熵税（为纠偏增加的摘要、记忆、检查、回滚机制本身消耗 token）。Luo et al. (2026) 观察到链式推理路径过长时容易陷入循环，对抗性注入可故意触发。一个看似高效的 Agent 团队，token 账单中可能超过一半花在内部协调与自我纠偏上。

这些损耗对 token 消耗的效果不是加和而是乘积性的。且影响不对称：技术背景开发者可通过调 system prompt、裁剪技能、管理上下文缓解；普通企业用户只看到账单增长却不知道钱花在哪。**token 不经济不仅是技术效率问题，更是技术平权问题**——AI 工具的使用门槛从"会不会写代码"变成了"能不能理解 Agent 架构的成本动力学"。

### 三、寻找真实的需求（应用端）

token 的有效使用大都局限在数字化水平较高的场景（编程辅助、文档处理、数据分析），性能随场景数字化水平下降而急剧衰减。

**编程是通用的特例**：通用性在于编程输出的是 Agent 的通用语言，可在数字化基础好的场景中直接驱动各类任务。特例在于编程在后训练环节具备极大优势：(1) 确定的信号反馈——编译器、解释器、单元测试立刻给出精准、结构化、无歧义的对错判断；(2) 自动的后训练闭环——反馈可无阻碍接进强化学习回路。这样的自主训练环境在其他场景基本不可能形成。

一旦离开编程，数据采集和结果验证成本会吃掉任何 token 经济。案例：A&O Shearman × Harvey AI，3,500 余名律师提交约 40,000 次查询，确实提效，但所有输出都需执业律师仔细审查后方可使用——资深合伙人复核 AI 标注合同的时间几乎等同于从头审阅。

**向物理世界的艰难跨越**：现实世界没有编译器，物理世界不接受迭代，只接受验证，而验证的成本永远比生成的成本高（Moravec 悖论仍在升值；李飞飞把空间智能与具身智能列为中期目标）。仿真技术撞上 Sim-to-Real Gap：OpenAI Dactyl 灵巧手在仿真中积累 13,000 年训练经验，面对真实材质、温度、磨损时鲁棒性迅速下降，2021 年 OpenAI 解散机器人团队。非对称验证成本从侧面说明编程场景的特殊性。

### 四、Token 不经济的溢出风险（产业链）

上游基础设施和硬件厂商赚得盆满钵满（2025 年台积电净利率 44.5%、英伟达 55.6%，自由现金流高速增长）；中游模型厂商营收爆发但深陷亏损；下游企业开始控成本。风险向中游聚集。

循环融资：OpenAI-英伟达-甲骨文资金闭环；OpenAI 算力采购框架加总突破 1 万亿美元，与 330 亿美元年化营收不匹配。Bain 测算：消化到 2030 年新增 200GW 算力需终端消费每年创造约 2 万亿美元新增营收，即便算上成本节约仍有约 8000 亿美元缺口。估值泡沫有一半藏在不透明的私人信贷市场（约 3 万亿美元规模，mark-to-model 定价、期限错配、PIK 利息滚入本金）。BIS：股权市场已充分定价上行空间，债务市场还没把下行风险定价进去。

资源饥渴挤压民生：弗吉尼亚数据中心巷耗电超州最大核电站两倍以上；Dominion 容量拍卖价格从 29 美元/MW-天飙升至 444 美元，涨幅超 1400%；居民电费账单可能翻倍。

### 五、寻找 Token 价值方程式（解决方案)

**技术面精细化变革**：

- 上下文缓存与语义压缩：context caching 在 Agent 路径高度分叉时缓存失效，更根本的解法是语义层面的主动压缩（保留关键指令和推理链路，去掉重复与冗余）。
- 技能优化与减法思维：SkillReducer 两条路径——描述压缩（为缺路由描述的技能补精简信息）+ 渐进式加载（按需加载），叠加后技能体压缩 39%，模型功能质量反而提升 2.8%。Less is more。
- 模型路由与任务分流：按任务复杂度自适应路由，简单高频子任务给开源轻量模型，关键决策点才用 Frontier 模型。
- 多 Agent 预算硬约束与主持人架构：为每个子任务设 token 预算上限，主持人 Agent 监控全局消耗，预算耗尽前强制终止无效循环。

**商业面价值锚定**：

- Token 治理与成本纪律：配额、审批、模型路由、成本归因、团队账单将成为企业 AI 治理基本方式。没有计量就没有管理，没有上限就没有纪律。真正先进的公司考核的不是"用 AI 最多"，而是"用最少的 token 完成最多的工作"。
- 配给制会成为常态：像管理云计算资源一样设定预算池和审批流程。配给制会倒逼架构师设计更高效的 Agent 系统，将成本约束内化。
- 找到 token 大规模商业应用的现实场景：务实路径是在传统行业中寻找具备弱确定性反馈的中间地带——辅助诊断中的影像筛查（有影像学标准）、供应链需求预测（有历史数据可回测）、法律合同初筛（有条款模板可比对）。这些场景验证成本虽不如编译器趋近于零，但远低于纯物理世界验证，有望成为 token 经济从数字沙盒走向真实世界的桥梁。

**回归 ROI**：今天的 token 和 Agent 架构正如早期的蒸汽机，噪音大、油耗高，但方向已现。未来更有价值的 Agent，不是思维链最复杂的 Agent，而是用最少 token 把事做成的 Agent。当每一枚 token 的消耗都必须回答产出有何价值，智能体时代就找到了自己的价值方程式。
