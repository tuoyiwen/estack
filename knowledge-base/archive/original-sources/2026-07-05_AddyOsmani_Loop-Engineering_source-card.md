# Source Card: Addy Osmani, "Loop Engineering"

- Author: Addy Osmani（Google Chrome 团队）
- Date: 2026-06-08
- Status: 未取全文，经 Loop Engineering 综述二手引用

## 核心主张

- Loop = 五个原语 + 状态存储：Automations、Worktrees、Skills、Connectors、Sub-agents（maker-checker 分离）、External State。Codex 与 Claude Code 已独立收敛到几乎相同的 Loop 架构。
- 两个关键风险概念：**Comprehension Debt**（Loop 越快产出你没写的代码，理解缺口越大且不报警）与 **Cognitive Surrender**（Loop 顺畅时人停止审查、质疑、思考）。
- "Build the loop. Stay the engineer."——用判断力设计 Loop 是加速器，用 Loop 逃避思考是灾难。

## estack 用途

L4 治理风险的一手出处；`$e-ai-governance` 讲自主化风险时引用。Intent Debt（意图负债）概念亦出于此。
