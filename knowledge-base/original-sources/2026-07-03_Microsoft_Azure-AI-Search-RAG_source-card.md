# Microsoft Azure AI Search RAG

## Archive Status

Source card only. Official documentation is linked, not copied.

## estack Relevance

Microsoft frames RAG as grounding model responses in proprietary content and distinguishes agentic retrieval from classic RAG for complex or conversational queries. For estack, this clarifies the boundary between public web search and internal enterprise retrieval.

## Extracted Use

- Separate public web search from internal proprietary-content retrieval.
- Use agentic retrieval when queries are complex, conversational, and need citations/query details.
- Add retrieval boundary decisions to workflow and governance templates.
